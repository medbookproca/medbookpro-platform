# Product documentation

This directory contains the product source of truth for MedBookPro. The master requirements document is implementation-aware, but it does not replace accepted architecture decisions, security review, legal advice, or migration review.

## Documents

- [MedBookPro Master Product Requirements Document](medbookpro-master-prd.md)

## Change control

Product requirements changes must be reviewed by product, engineering, security/privacy, and affected operational stakeholders. Changes that alter tenancy, authorization, data ownership, lifecycle, auditability, or integration trust boundaries require an ADR or an update to the relevant ADR before implementation. Requirements must remain clearly labelled as implemented, approved, planned, deferred, or open.
